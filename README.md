# 디싹 캠프 서류 자동화

캠프명단 엑셀을 업로드하면 정해진 양식 서류를 자동 작성하는 **브라우저 단독(서버 없음)** 웹앱.
모든 처리는 브라우저 안에서만 이루어져 개인정보가 외부로 나가지 않습니다. GitHub Pages로 배포.

## 생성 서류 (7종)

캠프명단 1개를 업로드하면 아래 서류를 자동 작성합니다.

| 서류 | 형식 | 단위 | 자동 채움 내용 |
|------|------|------|------|
| 수업신청학생등록양식 | xlsx | 클래스별(오전/오후) | 학생 **익명화**(권0구)·연락처·이메일·지역·학년·반·일반학생여부 / 드롭다운·텍스트서식 보존 |
| 수령대장 | hwpx | 클래스별 | 학생 **실명**·차시별 날짜 |
| 교구관리대장 | hwpx | 클래스별 | 프로그램·교육기간·운영일시(날짜+시간)·기관·학교·교구수량·주강사 (수령일시=시작일, 제출일=마지막일) |
| 결과보고서 | hwpx | 클래스별 | 프로그램·교육장소·**회차별 운영일시**·운영기간 (일수에 맞춰 회차 블록 자동 확장/삭제) |
| 안전업무일지 | hwpx | 클래스별 | 프로그램·운영일시·운영기관·**안전관리자 성명** |
| 안전체크리스트 | hwpx | 클래스별 | 점검책임자(안전관리자)·점검일자(운영전=시작일/운영후=마지막일) |
| 강사료지급신청서 | hwpx | **캠프 1부** | 프로그램·교육장소·교육대상·산출내역(오전+오후 합산)·**지급액=45,000원×차시**·신청일자=마지막일 |

### 변환 규칙
자세한 규칙은 [`docs/RULES.md`](docs/RULES.md) 참고.
- **학생명** 가운데 글자 `0` 익명화 (권준구→권0구) — 등록양식만. 수령대장은 실명.
- **연락처** `010-1234-1234` 고정 더미, 텍스트 서식
- **이메일** 임의 자동생성
- **지역** 학교명 자동완성: 번들 매핑 → 원DB → NEIS API → 수동
- **학년** `초등학교 N학년`, **반** 1~4 랜덤
- **일반학생 여부** 다문화/사회배려 학급(체크박스)에서 일반학생만 `Y`

### 사용법
1. `index.html`을 열거나 GitHub Pages 주소 접속
2. 캠프명단 xlsx 업로드 (표준 양식 내려받기 링크 제공)
3. **설정 패널**에서 클래스별 확인/수정:
   - 프로그램명(드롭다운, 자동선택) · 다문화/사회배려 학급(체크 시 확인질문)
   - 총 차시 · **일차별 날짜+시작/종료 시간**(일차 추가·삭제로 일수 가변) — 원DB에 일정 있으면 자동
   - 기관(대림대/가천대) · **주강사/보조강사/안전관리자** 이름(역할 자동) · 교구 수량
4. **변환** → 서류별 다운로드. 교구관리대장은 주강사 있을 때만 활성(보조강사 명단이면 비활성)
5. 지역은 학교명으로 자동 입력(번들→원DB→NEIS→수동). 미확인 학교는 미리보기에서 시·도 지정→재변환
6. 원DB(모집현황)는 비PII 참고데이터(`data/program_db.json`)로 내장돼 백그라운드 자동 로드(지역·차시별 일정)

### 지역 자동조회 (NEIS)
별도 키/설정 없이 동작합니다. NEIS 개방 API는 키 없이도 호출되고 CORS(*)도 허용되어
브라우저에서 학교명으로 전국 학교의 시·도를 바로 조회합니다.
호출량이 많아 한도에 걸릴 때만 `js/config.js`의 `NEIS_API_KEY`에 무료 키를 넣으면 됩니다(선택).

### 로컬 실행
모듈(ESM) 사용으로 `file://`이 아닌 로컬 서버가 필요합니다:
```
cd test
python -m http.server 8000
# http://localhost:8000
```

## 구조
```
index.html              UI (단계 카드 + 설정 패널 + 다운로드 그리드 + 미리보기)
styles.css
js/app.js               화면/이벤트 연결 · 설정 패널 · 다운로드 핸들러
js/config.js            NEIS API 키 설정(선택)
js/convert.js           명단 파싱 · 변환 규칙 · xlsx 템플릿 채우기 · 일정 파서
js/hwpx.js              hwpx 6종 생성(템플릿 셀/플레이스홀더 채우기, 회차 확장, 검정색 처리)
js/region.js            지역(시·도) 해석기 + 원DB 차시별 일정 조회
data/region_map.json    학교명 → 시·도 매핑 (누적)
data/program_db.json    원DB(모집현황) 비PII 참고데이터 — 지역·차시별 일정, 백그라운드 자동 로드
templates/              양식 파일(xlsx 2 · hwpx 5) — 채우기 대상(빈 양식, PII 없음)
samples/                테스트용 샘플 명단(.gitignore)
docs/RULES.md           변환 규칙 명세
```

## 안정성 메모
- hwpx는 `mimetype` STORE(무압축)+첫 엔트리 유지·나머지 DEFLATE·폴더 엔트리 제거로 한글 호환 보장.
- 드롭다운 MEMO 필드는 `fieldBegin/fieldEnd` 짝을 보존하며 텍스트만 비움.
- 채운 내용은 charPr 검정 복제본으로 교체해 글자색만 검정 처리(폼별 색 용도 차이 보존).
- ⚠️ hwpx 산출물은 실제 한글(HWP)에서 최종 확인 권장.

## 기술
순수 HTML/JS · [SheetJS](https://sheetjs.com) (xlsx) · [JSZip](https://stuk.github.io/jszip/) (xlsx/hwpx 템플릿 채우기).
복잡한 hwpx 처리가 필요하면 [edwardkim/rhwp](https://github.com/edwardkim/rhwp)(Rust+WASM, HWP/HWPX 읽기·쓰기) 검토.
