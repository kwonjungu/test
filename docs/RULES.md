# 변환 규칙 (캠프명단 → 서류 자동작성)

캠프명단 1개에서 클래스(오전/오후)별로 **① 수업신청학생등록양식(xlsx, 익명화)** 과
**② 수령대장(hwpx, 실명)** 두 서류를 생성한다.

## 입력
- **캠프명단 엑셀** (`캠프명단 (학교_일자_담당자).xlsx`)
  - 시트: `오전반`, `오후반` (각각 별도 명단), `Sheet1(삭제 금지)`(프로그램 마스터, 사용 안 함)
  - 학생표 헤더: `No | 이름 | 전화 | 학교 | 학년 | 교육시작일 | 교육종료일 | 메모(특이사항)`
  - 헤더 위쪽에 `교육 장소`, `프로그램명` 메타가 있음
  - `신청 교사 및 문의` 섹션 아래는 강사 정보 → 학생 명단에서 제외

## 변환 전 편집(설정 패널)
업로드 직후 클래스별로 자동 채워지며, **변환 버튼 전에 사용자가 수정 가능**:
- **다문화/사회적배려 학급 체크박스** — 체크 시 "명단 메모에 다문화 학생을 표시했는지" 확인창. 체크 여부가 `일반학생 여부 Y` 판정의 기준이 됨(프로그램명 자동감지보다 우선).
- **총 차시 / 일별 차시** — 과정 기본값(기본=8, 특화/AI특화=12)에서 시작, 수정 가능.
- **일자(날짜)** — `교육시작일~교육종료일`을 일별로 펼쳐 `6월 20일` 형식으로 표시, 수정 가능.

## 출력 (수업신청학생등록양식.xlsx)
**클래스(오전반/오후반)별로 파일을 분리**해 생성한다. 클래스 수 = 명단에서 학생이 있는 시트 수(원데이터 기준, 원DB의 오전반(명)/오후반(명)과 대응). 학생이 없는 클래스는 제외. 각 클래스 파일은 **개별로 다운로드**(ZIP으로 묶지 않음).

열: `학생명 | 연락처 | 이메일 | 지역 | 학교 | 학년 | 반 | 일반학생 여부`

| 열 | 규칙 |
|----|------|
| 학생명 | **익명화**: 가운데 글자를 `0`으로. 예) 권준구→권0구, 이다은→이0은. 2글자=뒷글자→0(김수→김0). 4글자 이상=가운데 전부 단일 `0`(김베냐민→김0민) |
| 연락처 | 익명화 고정 더미값 `010-1234-1234`로 통일. **텍스트 서식** |
| 이메일 | 임의 자동생성 (영문 5~8자+숫자 @ naver/gmail/daum/hanmail/kakao) |
| 지역 | 학교명으로 자동 결정: ①번들 매핑(data/region_map.json) → ②원DB(업로드 시) → ③NEIS API → ④수동 드롭다운. 표기는 드롭다운 17개 시·도와 동일 |
| 학교 | **전체 학교명** 그대로 (증안초등학교) |
| 학년 | 숫자 → `초등학교 N학년` 형식. 초/중/고 구분은 학교명 접미사(초등학교/중학교/고등학교)로 판정, 없으면 프로그램 학교급(초저·초고·중등·고등) |
| 반 | **1~4 랜덤** 정수 |
| 일반학생 여부 | 기본 공란. **다문화/사회적배려 학급(체크박스)** 에서, 메모에 다문화·배려대상 표시가 없는 일반학생에게만 `Y` |

## 양식의 작성 주의사항(셀 I1 메모, 원본 규칙)
1. 셀 서식은 항상 **텍스트**로 유지 (숫자/수식 금지 → 데이터 변형 방지)
2. 연락처는 `010-123-1234` 형식(하이픈)
3. 이메일 하이퍼링크 스타일 해제 불필요
4. 학교명은 전체학교명
5. 반은 숫자만
6. 일반학생 여부(사회적배려자 학급만): 일반학생이면 `Y`, 그 외 학급 기입 안 함

## 양식 내부 제약 (보존 대상)
- **학년(F열)**: 드롭다운 잠금("변경이 불가합니다") — 12개 값 고정
- **지역(D열)**: 드롭다운 17개 시·도
- 모든 셀 텍스트 서식 → 템플릿 채우기 시 `inlineStr`로 강제하여 보존

## 출력 ② 수령대장 (hwpx, 실명)
서명 대장이므로 **실명** 사용(가운데글자 0 익명화 안 함). 템플릿 `templates/수령대장양식.hwpx`.
- 양식에 8차시 표(일차 2칸)와 12차시 표(일차 4칸)가 있음 → 설정의 총 차시로 표 선택(수기서명 2개=8차시 / 4개=12차시).
- **성명**: 표의 `colAddr="1" rowAddr=2~26` 셀에 학생명 순서대로 주입.
- **날짜 헤더**: `colAddr>=2 rowAddr=0`의 `(N일차) 월 일`을 설정 일자(`6월 20일`)로 치환.
- 클래스(오전/오후)별 개별 hwpx로 다운로드.

### hwpx 패키징 규칙 (한글 호환)
- `mimetype`은 **STORE(무압축) + 첫 엔트리** 유지, 나머지는 DEFLATE.
- JSZip이 자동 추가하는 폴더 엔트리(`Contents/`)는 제거(원본에 없음).
- 빈 셀의 `<hp:run charPrIDRef="N"/>`에 `<hp:t>이름</hp:t>`을 주입(서식 ID 보존).

## hwpx 처리 방식 (라이브러리 검토)
의존성 없이 **템플릿 XML 문자열 치환**으로 처리(JSZip만 사용) — 정해진 양식의 셀 채우기에 충분하고 가볍다.
복잡한 hwpx 생성/렌더링이 필요해지면 [edwardkim/rhwp](https://github.com/edwardkim/rhwp)(Rust+WASM, MIT, HWP/HWPX 읽기·쓰기) 도입을 검토.
