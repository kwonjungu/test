# 디싹 캠프 서류 자동화 — 개발 기록

디지털새싹 캠프 제출 서류를 캠프명단(xlsx) 한 번 업로드로 자동 작성하는 웹앱.
이 문서는 **어떻게 만들었는지**(설계 결정·구조·핵심 로직)를 정리한다.

---

## 1. 두 배포본

| 레포 | 호스팅 | 성격 |
|---|---|---|
| `kwonjungu/test` | Vercel | 지인용. 서버리스 함수(AI 의견·공유 링크) 포함 |
| `kwonjungu/drdi` | GitHub Pages | 외부/공개용. **완전 클라이언트(서버 없음)**, 개인정보 저장 안 함 |

`drdi`는 `js/hwpx.js` 등 클라이언트 코드만 동기화하고 `api/`·Firebase 기능은 제외한다.

## 2. 전체 흐름

```
캠프명단(xlsx) 업로드
  → parseRoster (convert.js)         : 클래스(오전/오후)별 학생·학교·담당자 파싱
  → 설정 패널(2.5)에서 확인·수정      : 프로그램/차시/일정/주·보조·안전강사/교구
  → 변환하기
  → toRegistrationRows               : 익명화 등록행 생성
  → 서류별 build*Hwpx (hwpx.js)       : 템플릿 hwpx에 값 치환 → 다운로드
```

- 모든 처리는 **브라우저 안에서** 수행(xlsx/jszip은 CDN). 서류 생성은 클라이언트.
- Vercel 버전만 `/api/opinion`(Gemini AI 의견)·`/api/save·load`(Firebase 공유) 사용.

## 3. hwpx 처리 원리

hwpx는 **zip(OCF) + XML** 패키지. 본문은 `Contents/section0.xml`, 서식은 `Contents/header.xml`.

- `Contents/section0.xml` : `<hp:p>`(문단) > `<hp:run charPrIDRef="N">`(글자속성 참조) > `<hp:t>`(텍스트)
- `Contents/header.xml` : `<hh:charPr id="N">`(색·크기·폰트), `<hh:paraPr id="N">`(정렬·줄간격)

`js/hwpx.js`가 이 XML을 문자열로 열어 텍스트를 치환하고 다시 zip으로 패키징한다(`packageHwpx`).

## 4. 핵심 설계 결정 — "사용자 완성본을 마스터 템플릿으로"

### 초기 방식(폐기): 자리표시자 + 서식 강제 통일
처음엔 빈 양식의 자리표시자(`00초등학교`, `해당 프로그램명 작성` 등)를 채우고,
채운 글자를 검정·11pt·함초롬바탕으로 **강제 변환**(cloner)했다.
→ 양식 고유 서식과 충돌해 **라벨 겹침·파란 글씨·기울임·폰트 꼬임**이 반복 발생.

### 최종 방식: 완성본 마스터 치환
사용자가 한글에서 **직접 서식을 완성한 산출물**(폴더 `꾀꼬리/`)을 템플릿으로 채택하고,
거기 박힌 **실제값만 새 입력으로 문자열 치환**한다. 서식은 절대 건드리지 않는다.

```
마스터에 박힌 기준값(M)
  프로그램 = (기본/초저) 노벨엔지니어링으로 만드는 안전한 등굣길
  학교 = 증안초등학교 / 기관 = 대림대학교
  주강사 = 권준구 / 보조강사 = 원종민 / 안전관리자 = 이소정
  교구 = 25 / 날짜 = 6/20~6/21 / 시간 = 09:00~12:10(오전)·13:00~16:10(오후)
```

`applyMaster(xml, data)` 공통 헬퍼가 이 값들을 새 캠프 데이터로 치환한다:
- 프로그램/학교/기관/담당자/교구수량 → `replaceAllText`
- 날짜는 여러 형식 모두 대응: `2026년 06월 20일 ~ 06월 21일`(pad), `2026년 6월 20일 ~ 2026년 6월 21일`(서약서), `2026. 06. 20.`(dot), `2026.06.20`(수령일시), 단독 `2026년 6월 21일`(지급/체크리스트)
- 일차 단락은 `fillMasterDays`가 **일수(1~4일)에 맞게 복제/삭제**

> 효과: 서식 문제 원천 차단. "사용자가 손으로 맞춘 그대로" 나오고 캠프 데이터만 바뀜.

각 서류는 **개별 분석·검증**하여 빌더를 만들었다(패턴이 비슷해도 산출/금액/회차 구조가 달라 일괄 처리하지 않음).

## 5. 서류별 처리 요약 (`js/hwpx.js`)

| 서류 | 빌더 | 특이 처리 |
|---|---|---|
| 교구관리대장 | `buildEquipmentLedgerHwpx` | 일차·수령일시(시작)·제출일(마지막) |
| 수령대장 | `buildReceiptHwpx` | 학생 이름 빈 run 주입, 8/12차시 표 선택 |
| 안전관리 서약서 | `buildSafetyPledgeHwpx` | 현장안전담당 치환, 운영기관·PM 고정 유지 |
| 안전업무일지 | `buildSafetyLogHwpx` | 업무내용 30개 풀에서 **매번 랜덤 10개** |
| 안전체크리스트 | `buildChecklistHwpx` | 점검책임자 전 노드, 운영전=시작일·운영후=종료일 |
| 운영 사례집 | `buildCaseBookHwpx` | 후기 3개 AI 교체 + **이름(학생 랜덤1·학부모 생성·강사 익명)** |
| 주강사료 | `buildPayApplicationHwpx`(주 분기) | 산출 75,000×차시, 금액·날짜 |
| 보조강사료 | `buildPayApplicationHwpx`(보조 분기) | `■ 보조강사`로 구분, 산출 45,000×N차시×N회 |
| 안전 지급신청서 | `buildSafetyPayHwpx` | 일별차시 캡 60,000원, 금액 |
| 결과보고서 | `buildReportHwpx` | **회차 블록 1~4일 복제** + 추진의견 3개(AI) + 식다과/교재 수령대장 |
| 근로계약서·다문화 | `buildSafetyContract`·`buildMulticultural` | 완성본 미제공 → 기존 자리표시자 방식 |

### 금액 정산 규칙
- 주강사료: 75,000원 × 총차시
- 보조강사료: 45,000원 × 총차시
- 안전: 20,000원/차시, **반(班)별 1일 한도 60,000원**(=3차시). 일별 = min(차시×20,000, 60,000), 오전·오후 별개 합산
  - 예) 12차시·3일(4차시/일) → 일별 80,000을 60,000으로 캡 → 180,000원

## 6. AI 추진의견·후기 (Vercel 전용)

- `api/opinion.js` : Google **Gemini**(`gemini-2.5-flash`, 환경변수 `GEMINI_MODEL`로 변경 가능)
- 하네스: 모델 폴백 체인 + 타임아웃(25s) + 재시도 + 부분실패 허용 + 마크다운/AI어투 후처리
- `api/program-context.js` : 7개 프로그램 교안 요약을 프롬프트에 주입해 **환각 차단**(.hwp PrvText·hwpx에서 추출)
- 프롬프트 규칙: 별표(`**`)·머리기호·제목 금지, 공문체/증언체, 2026년·학교명 환각 금지
- 결과보고서(추진의견)·운영사례집(후기)에 **다운로드 시 자동 생성·삽입**, "AI 추진의견" 박스에서 복사도 가능

## 7. 공유 링크 (Vercel 전용)

- `api/save.js`·`api/load.js` : Firebase **Admin SDK**(서비스 계정). 브라우저는 Firestore에 직접 접근 안 함 → 보안 규칙 `if false` 그대로 유지
- 변환 후 코드 입력 → `https://<site>/<코드>` 링크 생성. 링크 열면 명단·지역·시간·차시·담당자까지 설정 복원 후 서류 다운로드
- 환경변수: `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, `FIREBASE_PRIVATE_KEY` (자세한 건 `SETUP_VERCEL.md`)

## 8. 주요 트러블슈팅

- **라벨 겹침·파란 글씨·폰트 꼬임** → 서식 강제 변환이 원인. 완성본 마스터 방식으로 전환해 해결(§4)
- **문서 보안 낮음에서만 열림** → 완성본에 포함된 빈 한글 스크립트(`Scripts/*.js`)가 "스크립트 포함 문서"로 분류됨. `packageHwpx`에서 `Scripts/*` 삭제 + `content.hpf`의 item·spine 참조 제거
- **결과보고서 추진의견 한 줄 겹침** → 빈 칸 paraPr의 고정 줄정보(linesegarray) 때문. 왼쪽정렬·줄간격160 paraPr로 교체 + linesegarray 제거
- **.hwp 교안 텍스트 추출** → 이미지 PDF라 OCR 불가, `.hwp`는 `olefile`로 PrvText 스트림 추출

## 9. UI 편의

- 서류 다운로드 버튼을 **역할별 그룹**(공통/주강사/보조강사/안전관리자) 색상·인덱스로 구분
- 담당자(주·보조·안전) 이름 입력 시에만 해당 서류 버튼 활성화
- 설정 패널 오후반에 **"⬆ 위와 동일"** 버튼: 오전반 설정(프로그램·차시·기관·담당자·교구) 복사
- 안전관리 이수증은 빈 양식 **항상 다운로드** 링크
